# Kestrel Documentation Hub

Kestrel is a Windows-native AI coding agent designed to feel instantaneous, deeply context-aware, and trustworthy on real engineering projects. The project goal is not to be another chat panel beside a code editor. Kestrel should become the fastest and most reliable way for Windows developers to understand, change, test, and ship software with AI.

## How to Read This Repository

There are two altitudes here, and both matter. The **[Horizon](./vision-horizon.md)** defines the summit: a decade-out future where the human owns intent and a fleet of local, verifying agents owns the perpetual labor of keeping systems correct. Everything else is the credible bridge to that summit — a buildable Phase 0 today, a verified-diff MVP next, and an architecture designed from the first commit to reach Era 3. Ambition without a path is fantasy; a path without ambition is just another tool. Kestrel insists on both.

## Product Thesis

Developers do not need a heavier editor, a slower terminal, or another AI wrapper that loses context after two prompts. They need an agent that understands the codebase structurally, works safely inside the local development environment, verifies its own changes, and stays cheap enough to use all day.

Kestrel wins by combining:

- Native Windows performance.
- Compiler-grade context rather than raw text dumping.
- Local-first indexing and execution.
- Verified diffs instead of untested suggestions.
- Transparent token and cost controls.
- Deep WSL, Docker, Git, and IDE integration.

## North Star

Kestrel should become the default AI engineering layer for serious Windows developers: fast enough to stay open all day, accurate enough to trust on production code, and extensible enough to support every major language, framework, repository shape, and team workflow.

The near-term metric is **verified engineering work completed per active developer per week.** Its horizon successor — the measure of the future we are building toward — is **correct system-behavior maintained per engineer, over time, at trusted autonomy.** See the [Horizon](./vision-horizon.md) for what that means and why it is where coding is going.
